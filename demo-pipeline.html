Demo Analysis Pipeline

Technical details of the cascaded two-level architecture used in the Surg-R1 software demo.

Cascaded Two-Level Architecture

The system processes laparoscopic cholecystectomy videos through two sequential stages. Surg-R1's frame-level outputs are fed directly into a window-level synthesizer as structured input — the two stages form a cascade, not parallel independent systems.

Stage 1: Frame-Level Perception Surg-R1

Every 3 seconds, Surg-R1 independently analyzes each sampled frame with three structured queries: surgical phase recognition, action description, and tool localization with bounding boxes. Each frame is processed without temporal context, producing raw per-frame annotations.

All R1 outputs (phase + action + tools) forwarded as input

Stage 2: Window-Level Synthesis Gemini

Every 15-second window, the synthesizer receives four types of input: (1) formatted R1 frame annotations (phase, action, and tool detections), (2) sampled images from the window (up to 10 frames), (3) historical context (global phase progression + recent 3 window summaries), and (4) dynamic phase constraints injected based on surgical history. It cross-references R1's annotations against its own visual understanding, resolves contradictions, and outputs a unified narrative summary with structured clinical metadata.

Key design principle: R1's outputs serve as the synthesizer's input, not as independent results. R1's tool detection is generally accurate and is directly incorporated. For phase recognition, R1 lacks cross-window context, so the synthesizer re-evaluates R1's phase predictions against the full surgical timeline. R1's perception is never discarded — it is validated and refined through a temporally-aware second pass.

Resolving Contradictions

Because Surg-R1 processes each frame independently, its per-frame predictions inevitably contain inconsistencies that must be resolved at the window level.

Intra-Window Contradiction

Frames within the same 15-second window may disagree on the surgical phase. For example, 3 of 5 frames might predict "GallbladderDissection" while 2 predict "CleaningCoagulation." The synthesizer receives all R1 frame annotations alongside the actual images, performs consistency analysis across the phase distribution, and determines the dominant phase by jointly weighing R1's predictions and its own visual evidence.

Inter-Window Contradiction

Since R1 has no memory across windows, it may predict early-stage phases even late in surgery. The synthesizer prevents regressions through three mechanisms: a global phase history that persistently tracks all phases ever reached, dynamic constraints injected into the prompt (e.g., after "GallbladderPackaging," only "CleaningCoagulation" is permitted), and a clinically-grounded phase transition table that defines all valid state transitions for laparoscopic cholecystectomy.
